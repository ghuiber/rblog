I gave up on Anaconda R

September 3, 2017

In March of this year I decided to run R and RStudio from inside an Anaconda installation as described here. That experiment is over. Two annoyances cropped up:

  1. My Anaconda RStudio wouldn’t shut down a running Shiny app. I reported this in all the obvious places: here, here and here. Evidently, it’s not a big enough problem to be worth anybody’s trouble. That is fine. In retrospect, it wasn’t that big a deal for me either.
  2. This one is though: the latest update of the conda kit (to R 3.4.1 and RStudio 1.0.153) didn’t fix the first issue, and also brought up the Error in tools::startDynamicHelp() : internet routines cannot be loaded message at launch time. This means that you won’t get online R help. Any attempt to ?? anything will repeat this same message.

So I’m back to a stock RStudio and a stock R downloaded as Mac OS X binaries. This pairing works as expected. I kept Anaconda because I may still nose around Python 3 at some point. If you do this and keep your Anaconda R, you will have to direct RStudio to the /usr/local/bin/R explicitly.

There are a couple of ways to do this. One is indirectly explained here (it actually shows how to point RStudio to an Anaconda R but the recipe launchctl setenv RSTUDIO_WHICH_R can be changed to suit).

You could also simply push the Anaconda stuff to the back of the path; it puts itself at the front when you install Anaconda by writing a one-line ~/.bash_profile file that says

export PATH="/Users/<YOURUSERNAME>/anaconda3/bin:$PATH"

You can reverse the order and save over:

export PATH="$PATH:/Users/ghuiber/anaconda3/bin"

Or you could decide that you don’t need two R’s. You can get rid of the one under Anaconda with conda remove r-essentials and then conda clean. That is what I did.
